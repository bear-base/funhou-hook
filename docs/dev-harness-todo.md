# AI開発ハーネス セットアップTODO

分報システム開発にあたっての環境整備。一気にやらず、1つずつ入れて様子を見る。

---

## Step 1: 最低限（開発開始前に）

- [ ] リポジトリ作成・初期化
- [ ] TypeScript + tsconfig 設定
- [ ] CLAUDE.md 作成（プロジェクト概要、ディレクトリ構成の意図、技術選定の理由）
- [ ] lint / format 設定（ESLint + Prettier、自動適用）
- [ ] .claude/settings.json（基本的なpermission設定）

## Step 2: 壊れ防止（コア実装が始まったら）

- [ ] テスト環境セットアップ（vitest等）
- [ ] コアロジックのテスト（危険度判定、メッセージフォーマット）
- [ ] git commitを細かく切るルールをCLAUDE.mdに記載
- [ ] 型定義の整備（メッセージフォーマットの型）

## Step 3: 開発効率（手戻りが気になり始めたら）

- [ ] Skills定義（プロジェクト固有の判断基準）
- [ ] Hooks設定（テスト自動実行、format自動適用）
- [ ] デバッグ用の設定（ログ出力、環境変数）

## Step 4: 品質向上（動くものができてから）

- [ ] テストカバレッジの基準設定
- [ ] CI設定（GitHub Actions、push時にテスト+lint）
- [ ] ADR導入（大きな設計判断が出てきたら記録）

## Step 5: 必要に応じて

- [ ] PRフロー（協力者が増えたら）
- [ ] issue管理（TODOリストで足りなくなったら）
- [ ] 詳細な設計書（CLAUDE.mdで足りなくなったら）

---

## 判断基準メモ

- 迷ったら入れない。困ってから入れる
- 「AIに任せるからこそ必要」なものを優先（型、lint、テスト）
- 仕組みを作ることが目的にならないよう注意
- 各Stepで一度動かして、効果を確認してから次へ
